Projects Page Implementation Guide

This guide shows how to use the captions, alt text, and schema markup for Google SEO.

1. Basic Image with Alt Text & Caption

Simple implementation:

<figure>
  <img
    src="assets/projects/project-1/webp/01-IMG-20160109-WA0014.webp"
    alt="Existing industrial crane and equipment frame showing aged infrastructure requiring modernization"
  >
  <figcaption>Existing Equipment Assessment - Determining Scope of Work</figcaption>
</figure>

2. Image with Schema Markup (Recommended)

SEO-optimized implementation:

<figure itemscope itemtype="https://schema.org/ImageObject">
  <img
    src="assets/projects/project-1/webp/01-IMG-20160109-WA0014.webp"
    alt="Existing industrial crane and equipment frame showing aged infrastructure requiring modernization"
    itemprop="url"
    loading="lazy"
    width="1200"
    height="800"
  >
  <figcaption itemprop="name">Existing Equipment Assessment - Determining Scope of Work</figcaption>
  <meta itemprop="description" content="Initial site assessment of aging equipment and infrastructure to determine project scope. Shows weathered structural framework and lifting equipment that will be replaced or refurbished.">
  <meta itemprop="datePublished" content="2016-01-09">
  <meta itemprop="author" content="Q2 Machines">
  <meta itemprop="keywords" content="equipment assessment, site inspection, industrial equipment, crane, structural frame">
</figure>

3. Project-Level Schema (Add to Page Head)

Add this once per projects page in <head> or before </body>:

<script type="application/ld+json">
{
  "@context": "https://schema.org",
  "@type": "Project",
  "name": "Industrial Plant Modernization & Equipment Upgrade",
  "description": "Complete plant upgrade spanning 9 months from decommissioning through full operational commissioning",
  "image": [
    "https://q2m.io/jobs/assets/projects/project-1/webp/01-IMG-20160109-WA0014.webp",
    "https://q2m.io/jobs/assets/projects/project-1/webp/29-20160819_065320.webp"
  ],
  "startDate": "2016-01-09",
  "endDate": "2017-03-06",
  "location": {
    "@type": "Place",
    "name": "Trinidad and Tobago"
  }
}
</script>

4. Gallery/Collection Schema (For Multiple Projects)

If you have multiple project galleries on the page:

<script type="application/ld+json">
{
  "@context": "https://schema.org",
  "@type": "CollectionPage",
  "name": "Q2 Machines Project Gallery",
  "description": "Case studies of completed industrial equipment and plant modernization projects",
  "url": "https://q2m.io/jobs/projects",
  "image": {
    "@type": "ImageObject",
    "url": "https://q2m.io/jobs/assets/projects/project-1/webp/29-20160819_065320.webp"
  }
}
</script>

Key SEO Benefits

Alt Text Benefits:

  • Helps Google Image Search index your images
  • Improves accessibility for screen readers
  • Appears if image fails to load
  • Includes relevant keywords naturally

Visible Captions Benefits:

  • Improves user engagement and time-on-page
  • Provides context for the content
  • Helps Google understand image subject
  • Improves click-through rate from search

Schema Markup Benefits:

  • Rich results in Google Search
  • Structured data helps Google understand relationships
  • May show in Google Images with extra metadata
  • Improves SEO for project showcase/portfolio pages

Implementation Checklist

All images use WebP format (original JPG kept in archive) Each image has descriptive alt text (125+ chars) Each image has visible caption using <figcaption> Images use <picture> element with WebP + JPEG fallback (optional but recommended) Schema markup added for project-level data Images have width/height attributes to prevent layout shift Images use lazy loading (loading="lazy") Alt text matches description from captions.json

Testing & Validation

  • Google Rich Results Test: https://search.google.com/test/rich-results
  • Schema.org Validator: https://validator.schema.org/
  • Google Images: Test that images appear in Google Image Search
  • Lighthouse: Check accessibility score (alt text coverage)
